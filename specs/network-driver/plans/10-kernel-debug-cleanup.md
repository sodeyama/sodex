# Plan 1.10: 起動経路のデバッグ整理

## 目的

`kernel.c` に残っている bring-up 用の一時デバッグを、本番経路から切り離す。

## 完了メモ

この plan は完了。

- `src/kernel.c` から NE2000 / PIC の一時ダンプを削除
- 手動 ARP 送信や IRQ 監視ループを削除
- 通常起動では `init_ne2000()` -> `uip_init()` -> `network_init()` の後に通常ループへ入る形へ戻した
- 通常起動経路は `make -C src test-qemu-shell-io` で確認した

## 現状

`src/kernel.c` に以下が混在している。

- NE2000 レジスタダンプ
- PIC ダンプ
- ARP request の手動送信
- 送受信完了待ちループ
- IRQ カウンタの確認

これらは初期 bring-up では有効だったが、通常起動経路としては重い。

## タスク

1. 常用コードと検証コードを分離する
2. 一時デバッグを `KTEST_BUILD` などに寄せる
3. 通常起動では `network_init()` 後に素直にサービスループへ入る形に戻す

## 方針

- 起動確認用の観測は `ktest` または専用デバッグフラグへ移す
- `kernel.c` は最小の初期化と通常ループに寄せる

- [x] 通常起動のフローが読みやすい
